# Массовая загрузка товаров и фото

## Логика

Одна строка в шаблоне = один основной артикул / один принт. Сайт сам создает варианты по столбцам:

- `Основной артикул`: для папок `opt_67895` пишем `opt_67895`, без удаления префикса `opt_`
- `Типы товара`: например `Подушка; Наволочка`
- `Категории`: например `Подушки; Наволочки`
- `Размеры`: например `30x30; 35x35; 40x40; 45x45; 50x50`
- `Материалы`: например `Велюр; Габардин`

В CSV-файлах разделитель колонок тоже `;`. Внутри ячеек со списками несколько значений также разделяются через `;`, поэтому Excel будет брать такие ячейки в кавычки автоматически. Старые файлы с запятыми в ячейках тоже читаются, если в ячейке нет `;`, но новые файлы лучше вести через `;`, чтобы размеры вроде `3,5` не ломались.

Правило артикула варианта: `Основной артикул_ТИП_Размер_МАТ`, где разделитель `_`, тип товара и материал сокращаются до первых 3 символов после очистки, а размер пишется полностью. Примеры: `Подушка` -> `ПОД`, `Наволочка` -> `НАВ`, `Велюр` -> `ВЕЛ`, `Габардин` -> `ГАБ`, `Oxford` -> `OXF`.

Если один принт должен попадать в две категории, в колонке `Категории` пишем их через `;`: `Подушки; Наволочки`. В колонке `Типы товара` для такого товара можно написать `Подушка; Наволочка`. Если название базовой карточки начинается с `Подушка`, то вариант с типом `Наволочка` на сайте будет отображаться как `Наволочка ...` автоматически.

Каждая комбинация `тип × размер × материал` считается отдельным артикулом варианта. В админке можно скачать отдельную таблицу цен вариантов: там каждая строка = один вариант с колонками `Основной артикул`, `Артикул варианта`, `Тип товара`, `Размер`, `Материал`, `Цена варианта`. Это нужно для редактирования цены каждого варианта отдельно. Для текущей партии локально также формируется файл `local-import-output/variant-prices.xlsx`.

Фото ищутся по папке основного артикула. Пример:

```text
Фото товаров/
  10345/
    1.jpg
    2.jpg
    3.jpg
  10346/
    1.png
    2.webp
```

Также поддерживается вложенная структура `категория / артикул / фото`. В этом случае сканер берет категорию из предпоследней папки, основной артикул из последней папки, приводит его к виду `opt_число`, а относительный путь записывает в колонку `Папка фото`.

```text
Фото товаров/
  пляжные сумки/
    opt_Акулы_68029/
      1.jpg
      2.jpg
  шевроны 8х10/
    48100/
      1.jpg
```

`1.jpg` обычно становится главным фото, остальные уходят в галерею. Лучше держать фото квадратными `1200x1200 px`, JPG/WebP.

## Если список артикулов уже есть

Скачать шаблон можно из админки кнопкой `XLSX-шаблон` или из файла:

```text
templates/sobag-products-template.xlsx
```

После заполнения можно загрузить таблицу через админку или прогнать локальный импортер.

## Если списка артикулов нет

Импортер умеет собрать черновой шаблон из названий папок с фото:

```powershell
python tools/product_importer.py scan-photos --photos "C:\Path\YandexDisk\Фото товаров" --out local-import-output\products-from-photo-folders.xlsx
```

После этого файл `products-from-photo-folders.xlsx` можно открыть, дописать названия, категории, типы товара, подборки, праздники, цены и загрузить обратно.

Если поле `Подборки` пустое, товар не добавляется ни в одну подборку. Новые категории из таблицы добавляются на сайт автоматически при импорте через админку; временную иконку и описание можно поменять в админке позже.

При загрузке новых товаров старые товары не удаляются. Система проверяет `Основной артикул`: если такой артикул уже есть, строка считается дублем и пропускается. Повторная загрузка того же файла не должна создавать копии товаров.

## Локальный импорт

```powershell
python tools/product_importer.py import --table local-import-output\products-from-photo-folders.xlsx --photos "C:\Path\YandexDisk\Фото товаров"
```

По умолчанию локальный импортер тоже сохраняет старые товары в `data/products.import.json` и добавляет только новые артикулы. Дубли попадают в отчет `data/import-report.csv` со статусом `Дубль пропущен`. Полностью заменить выходной JSON можно только явной командой с флагом `--replace`.

Результат:

- `assets/imported-products/` - скопированные фото;
- `data/products.import.json` - подготовленные товары для будущего подключения к сайту/backend;
- `data/import-report.csv` - отчет по строкам и фото.

Эти выходные папки сейчас добавлены в `.gitignore`, чтобы случайно не загрузить тысячи фото в GitHub.

## Яндекс Диск

Самый удобный вариант: установить приложение Яндекс Диска и синхронизировать папку с фото на компьютер. Тогда для импортера это обычная локальная папка, например:

```powershell
--photos "C:\Users\SoBag\YandexDisk\Фото товаров"
```

Давать пароли или токены в чат не нужно. Если позже понадобится импорт напрямую по ссылкам/API, лучше подключать это отдельным безопасным способом.

## Проверка перед публикацией

Сейчас `data/products-live.json` хранит базовые карточки: один основной артикул = один принт/товарная карточка. Варианты для сайта генерируются автоматически из `Типы товара`, `Размеры` и `Материалы`. Поэтому в самом JSON массив `variants` может отсутствовать, а в админке и карточке товара варианты все равно будут рассчитаны.

Перед публикацией большой партии нужно запускать:

```powershell
npm run check
```

Эта проверка теперь дополнительно контролирует структуру товаров:

- основной артикул не пустой и не повторяется;
- обязательные списки `categories`, `types`, `sizes`, `materials` заполнены;
- внутри списков нет дублей с разным регистром;
- все фото существуют на диске;
- тестовые изображения и старые `SB-...` артикулы не попали обратно;
- все рассчитанные артикулы вариантов уникальны;
- цены вариантов получаются положительными.

Для быстрой проверки интерфейса можно запустить локальный сервер и smoke-тест:

```powershell
python -m http.server 4173 --bind 127.0.0.1
$env:SOBAG_BASE_URL='http://127.0.0.1:4173'
npm run ui:smoke
```

Локальный Python-сервер не поднимает `/api`, поэтому smoke-тест специально игнорирует локальные 404 на API и проверяет статическую витрину: каталог, избранное, фильтры, карточку, смену варианта, добавление в корзину и поиск.
